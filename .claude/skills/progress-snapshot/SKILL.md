# Progress Snapshot Generator

## Overview

Generate shareable, parent-friendly progress reports that synthesize student data into narrative summaries. Unlike `student-export` (raw data), this produces polished reports suitable for parent communication.

## Usage

When invoked, gather these inputs:

1. **Student** -- name, ID, or "all active" for bulk generation
2. **Date range** -- default: last 30 days. Options: "this month", "this quarter", "all time", or custom range
3. **Format** -- markdown (default), email, PDF
4. **Comparison** -- include peer comparison context? (default: no)

## Execution Steps

### Step 1: Fetch Student Data

```sql
-- Student profile
SELECT id, email, "firstName", "lastName", created_at
FROM profiles WHERE id = '<student_id>';

-- Song repertoire with status counts
SELECT
  status,
  COUNT(*) as count,
  array_agg(json_build_object('title', s.title, 'author', s.author, 'level', s.level)) as songs
FROM student_repertoire sr
JOIN songs s ON s.id = sr.song_id
WHERE sr.student_id = '<student_id>'
GROUP BY status;

-- Lesson attendance in date range
SELECT
  COUNT(*) FILTER (WHERE status = 'completed') as completed,
  COUNT(*) FILTER (WHERE status = 'cancelled') as cancelled,
  COUNT(*) FILTER (WHERE status = 'rescheduled') as rescheduled,
  COUNT(*) FILTER (WHERE status = 'scheduled') as scheduled,
  COUNT(*) as total
FROM lessons
WHERE student_id = '<student_id>'
  AND date >= '<start_date>' AND date <= '<end_date>';

-- Recent lesson notes (last 5)
SELECT date, notes, status FROM lessons
WHERE student_id = '<student_id>' AND notes IS NOT NULL
ORDER BY date DESC LIMIT 5;

-- Songs mastered in period
SELECT s.title, s.author, sr.updated_at
FROM student_repertoire sr
JOIN songs s ON s.id = sr.song_id
WHERE sr.student_id = '<student_id>'
  AND sr.status = 'mastered'
  AND sr.updated_at >= '<start_date>';

-- Assignment completion rate
SELECT
  COUNT(*) FILTER (WHERE status = 'completed') as completed,
  COUNT(*) as total
FROM assignments
WHERE student_id = '<student_id>'
  AND created_at >= '<start_date>';
```

### Step 2: Calculate Metrics

```
Attendance rate = completed_lessons / total_lessons × 100
Mastery rate = mastered_songs / total_songs × 100
Assignment completion = completed_assignments / total_assignments × 100
Progress velocity = songs_status_changed / weeks_in_period
```

### Step 3: Optional Peer Comparison

Use `lib/services/cohort-analytics.ts` for comparative context:

```typescript
import { getCohortAnalytics } from '@/lib/services/cohort-analytics';

// Compare mastery rate vs cohort
const cohort = await getCohortAnalytics('status', 'mastery_rate');
// → "Your student's mastery rate (45%) is above the cohort average (32%)"
```

### Step 4: Generate Narrative

Build a parent-friendly narrative from the data. Tone: encouraging, specific, forward-looking.

## Output Template

```markdown
# Progress Report: {Student Name}
**Period**: {start_date} - {end_date} | **Generated**: {today}

## Summary

{Student FirstName} has had a {positive adjective} {period_length}! They attended
{completed}/{total} lessons ({attendance_rate}%) and {mastery_highlight}.

## Attendance

| Metric | Count |
|--------|-------|
| Lessons completed | {completed} |
| Cancelled | {cancelled} |
| Rescheduled | {rescheduled} |
| Attendance rate | {rate}% |

## Song Progress

{Student FirstName}'s current repertoire of {total_songs} songs:

| Status | Count | Songs |
|--------|-------|-------|
| Mastered | {n} | {song_list} |
| Remembered | {n} | {song_list} |
| Started | {n} | {song_list} |
| To Learn | {n} | {song_list} |

{If songs mastered this period:}
**New this period**: {FirstName} mastered "{song_title}" by {author}!

## Highlights

- {Achievement or milestone from lesson notes}
- {Technique improvement observed}
- {Consistency or effort recognition}

## Areas of Focus

- {Current technique being developed}
- {Song(s) actively being learned}
- {Theory concept being introduced}

## Recommended Next Steps

1. {Specific, actionable recommendation}
2. {Practice suggestion with time estimate}
3. {Goal for next period}

---
*Report generated by Strummy | {teacher_name}*
```

## Bulk Generation

When "all active" is selected:

1. Query all active students: `SELECT id FROM profiles WHERE "isStudent" = true AND "isActive" = true`
2. Generate individual reports for each
3. Output as concatenated markdown or individual files
4. Summary table at the top: all students ranked by key metrics

## Narrative Tone Guidelines

- **Always positive-first**: lead with achievements, then areas for growth
- **Specific over generic**: "Maria mastered 'Wonderwall' chord transitions" not "Good progress"
- **Quantified**: include numbers (attendance %, songs count, weeks practicing)
- **Forward-looking**: end with clear next steps
- **Parent-appropriate**: avoid jargon, explain musical terms

## Error Handling

- **No lessons in period**: "No lessons recorded in this period. Consider adjusting the date range."
- **No songs**: "No songs assigned yet. A repertoire will be built in upcoming lessons."
- **Student not found**: suggest searching by partial name or listing students
- **Notification service unavailable**: output markdown only, skip email delivery
